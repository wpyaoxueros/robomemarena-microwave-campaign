# Task22 v16 Cancelled Diagnostic: Later Targets Still Entered Hold

## What Worked

- The 35999 checkpoint again loaded the built-in norm asset with no fallback.
- `pick tomato` completed the official lift stage at step `156`.
- Its EEF hold began at step `177`.
- Because strict immediate-next release was disabled, the VLM's own
  `place tomato aside` candidate released that hold and drove 50 VLA actions.
  No prompt was generated by the runtime.

## Why It Was Stopped

The forward-gate selection list did not restrict which primitives can create an
end-pose hold. `place tomato aside` itself reached its target and entered hold
at step `482`. The VLM then returned `pick tomato`; the past-transition guard
correctly rejected that backward candidate, leaving the image static and
preventing a new decision.

The run was cancelled at scheduler elapsed time `00:06:29` rather than wasting
the remaining rollout budget. It is not a final Task22 result.

## Consequence

The next variant must restrict the *hold target set itself* to `pick tomato`.
That preserves the useful initial EEF gate while leaving all later prompt
transitions and actions under VLM control.
